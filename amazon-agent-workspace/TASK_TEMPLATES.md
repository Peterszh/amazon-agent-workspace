# 任务输入模板

为了确保任务顺利执行，请按以下格式提供信息。

---

## Listing撰写任务 输入字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 站点 | ✅ | 如：US/UK/DE/JP |
| 产品名称 | ✅ | 产品名称或型号 |
| 品类 | ✅ | 产品所属品类 |
| 品牌 | ✅ | 品牌名 |
| 核心卖点 | ✅ | 产品的核心优势 |
| 目标人群 | ✅ | 主要目标用户群体 |
| 竞品ASIN | ❌ | 可选，用于对标参考 |
| ABA关键词数据 | ❌ | 可选，ABA排名数据 |
| 广告搜索词数据 | ❌ | 可选，广告Search Term报告 |
| 当前标题 | ❌ | 如有现有Listing |
| 当前五点 | ❌ | 如有现有Listing |
| 当前描述 | ❌ | 如有现有Listing |
| 目标语言 | ✅ | 如：英语/日语/德语 |

---

## 亚马逊运营任务 输入字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 站点 | ✅ | 如：US/UK/DE/JP |
| 店铺/品牌 | ✅ | 店铺或品牌名 |
| ASIN/SKU | ✅ | 产品ASIN或SKU |
| 分析周期 | ✅ | 近7天/30天/指定日期 |
| 销量数据 | ✅ | 销售报表数据 |
| 流量数据 | ✅ | 流量报表数据 |
| 转化数据 | ✅ | 转化率数据 |
| 广告数据 | ✅ | 广告报表数据 |
| 价格与促销数据 | ✅ | 价格、促销、优惠券数据 |
| 领星ERP利润数据 | ❌ | 可选，ERP利润数据 |
| 库存数据 | ✅ | 库存周转、缺货风险数据 |
| 退款/退货数据 | ❌ | 可选，退款退货数据 |
| 备注 | ❌ | 最近是否有调价/断货/改图/活动 |

---

## 输出分层规则

**每个Agent必须强制区分三层：**

| 层级 | 说明 |
|------|------|
| **数据事实** | 基于实际数据能确认的事实 |
| **推测原因** | 基于数据的可能原因（标注为推测） |
| **建议动作** | 具体的可执行动作 |

---

## 缺数据时的默认行为

| 缺什么数据 | 默认行为 |
|------------|----------|
| 没有ABA数据 | 只做基础关键词结构建议，不做ABA排名分析 |
| 没有广告数据 | 只做静态Listing优化建议，不做广告词布局 |
| 没有ERP利润数据 | 不能对利润做强结论，只分析毛利层面 |
| 没有库存数据 | 不评估库存风险，只分析销售层面 |
| 数据都不全 | 明确标注"信息不足，只能做基础分析" |

---

## 优先级规则

**所有运营建议必须分优先级：**

| 优先级 | 说明 | 处理时间 |
|--------|------|----------|
| **P0** | 立刻处理，紧急问题 | 立刻 |
| **P1** | 本周处理，重要但不紧急 | 本周 |
| **P2** | 后续优化，长期改进 | 后续 |

## 缺口识别规则

当用户输入信息不足时，木棉Agent应先识别缺口，只问最少量问题：

**Listing任务必填缺口：**
- 无站点 → 询问目标站点
- 无产品名称 → 询问产品名称
- 无核心卖点 → 询问核心卖点

**运营任务必填缺口：**
- 无ASIN → 询问ASIN
- 无分析周期 → 询问想看什么时间段
- 无具体问题 → 询问想重点分析什么
